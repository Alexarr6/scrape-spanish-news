## Architect handoff notes

### Diagnosis

The current UI has one structural root cause for all its weaknesses: `AppShell.tsx` imposes a generic `filters / main / detail` three-column template on both routes, making Stories and Explorer feel like the same screen with different data injected. Everything else (chip overuse, weak card hierarchy, over-rounded surfaces, sidebar noise) is downstream of this template problem.

### Key decisions made

**Shell:** Deleted. Replaced by a lean `Shell.tsx` (top bar + main wrapper) and `TopBar.tsx` (wordmark + nav + scope chip). Sidebar gone. Both routes own their layout structure internally.

**Stories anatomy:** stream column + focus panel side by side. Filter drawer (slide-over) replaces the permanent left filter panel. The three-column template does not apply here.

**Explorer anatomy:** horizontal control bar above canvas + canvas (dominant) + context rail (320px). Control bar pulls all controls out of the floating canvas overlay. Canvas is clean semantic signal only.

**Coverage bar:** New component in StoryFocusPanel. Computed client-side from existing `members` array — no new API needed. Highest-value new element in the Stories rebuild; makes source coverage readable as a signal rather than a chip pile.

**Explorer context rail:** Tabs removed; replaced with divider-separated sections. Less procedural for this content volume. "Open in Stories" cross-link added (uses existing `cluster_id` on `ExplorerPoint.analysis`).

**API contract:** Unchanged for initial builder pass. All rebuilt surfaces work against existing endpoints. Optional backend improvements documented in UI_SPEC.md §8.2 for a future pass if needed.

**Visual system critical changes:**
1. Surface hierarchy: three levels only (page / panel / inset). The current CSS applies the same border+shadow stack to ~25 element classes — breaks hierarchy.
2. Border radius reduced: `1rem → 0.625rem` for panels, `0.85rem → 0.375rem` for inputs. Too-soft radius undercuts analytical character.
3. `.badge` system replaces dual `.status-chip` / `.summary-pill` which were functionally identical and overused.
4. Categorical color palette defined for map encoding.

### Implementation notes

1. **Shell contract broken.** `App.tsx` no longer passes shell as prop to routes. Routes own their layout entirely.

2. **MapPanel now forwardRef.** `ExplorerPage` holds a `mapRef` and calls `mapRef.current?.fitAll()` / `focusSelected()`. Controls live in `ExplorerControlBar`, not inside the canvas chrome.

3. **CoverageBar computed client-side.** Groups `members` by source, computes percentages, renders bar rows. No backend endpoint needed.

4. **FilterDrawer shared.** Both routes use the same `FilterDrawer` component. Filter fields are rendered as children (`StoriesFilterFields` and `ExplorerFilterFields` defined inline in their route files).

5. **ExplorerContextRail — tabs removed.** Content sections divided by `SectionDivider` labels. Guide / legend / dataset summary in no-selection state. Article / cluster / neighborhood in selection state.

6. **Stories focus panel — section architecture.** Brief → Coverage (CoverageBar) → Articles by source → Article detail (conditional on selection). Back button clears `selectedArticleId` without clearing story selection.

7. **"Open in Stories" link.** `ExplorerContextRail` calls `buildStoriesHref(cluster_id)` — navigates to `/?cluster=N`. The URL state hook reads `cluster` on mount, auto-selects the story. Cross-link is functional.

8. **Visual system.** `styles.css` fully rewritten. Design tokens as CSS custom properties. Three surface levels enforced (page / panel / inset). Border radius reduced. `.badge` replaces `.status-chip`/`.summary-pill`. No legacy sidebar styles.

### Risks and known gaps

- **Map canvas height:** The canvas height in `explorer-layout` depends on `flex: 1` cascading. If the outer `.app-main` does not fill viewport height correctly on some browsers, the canvas may render at zero height. CSS handles this with `min-height: 0` on flex children, but should be verified in a real browser.
- **Explorer context rail on tablet/mobile:** At <1280px, `context-rail` drops below canvas (grid stacks). The UX is functional but not polished for mobile — per spec, mobile is not the primary target for this iteration.
- **StoryFocusPanel sticky top:** Uses `position: sticky` with `top: calc(var(--topbar-height) + var(--space-4))`. Works correctly in a standard layout; verify if any parent gains `overflow: hidden` or `overflow: auto` which would break sticky.
- **Source color palette:** `SOURCE_COLORS` in MapPanel uses source slug keys. These need to match the actual source keys returned by the API. The old palette was also keyed this way — not a regression.
- **Optional backend upgrades** documented in UI_SPEC.md §8.2 remain un-implemented (dataset scope metadata for top bar chip, server-side coverage percentages). These are optional improvements for a follow-up pass.
